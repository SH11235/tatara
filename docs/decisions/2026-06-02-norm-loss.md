# Norm loss (per-weight-group L2-norm 正則化) の導入

- Status: Accepted
- Date: 2026-06-02

## Context

nnue-pytorch との Elo gap を詰める init / loss alignment の延長で、未 port の
正則化軸として norm loss (Georgiou et al. 2021, "Norm Loss: An efficient yet
effective regularization method for deep neural networks") を評価したい。tatara は
lr / WRM scaling / fanin init を nnue-pytorch と揃えており、norm loss は
Ranger21 由来で残っていた数少ない未実装軸。

参照実装は `nnue-pytorch`／そのフォークの `ranger21.Ranger21` (lessw2020/Ranger21
移植) と `model/lightning_module.py` の optimizer 構築。

## Decision

`--norm-loss` (default OFF) で opt-in する per-weight-group の L2-norm 正則化を
LayerStack trainer の optimizer step に追加する。強度は `--norm-loss-factor`
(既定 1e-4) で設定する。default OFF では norm loss kernel を一切 launch せず
baseline と bit-identical。

### 式

各 weight group `g` (= 1 出力ニューロンに流れ込む重みベクトル、または bias
テンソル全体) について step ごとに 1 回:

```
norm_g = ||w_g||_2
corr_g = 2 * factor * (1 - 1 / (norm_g + eps))
w_g   *= 1 - lr * corr_g
```

`norm_g > 1` は縮小、`< 1` は拡大で、両側から `norm_g = 1` (oblique manifold)
へ緩く引き寄せる。`lr` は schedule 後の実 lr、`eps` は optimizer eps を流用
(div-by-zero ガードのみで、norm が O(1) のため値の差は無視できる)。

### 対象層 (reference 準拠)

nnue-pytorch は `Ranger21` の `normloss_active=True` (factor 1e-4) を **全 param
group** に適用する。`unit_norm` がテンソル rank で粒度を決める: 2D linear weight
は `dim=1` (出力ニューロンごとの per-row)、1D bias はテンソル全体の scalar norm。
`StackedLinear = nn.Linear(in, out*count)` で weight は 2D `[out*count, in]`、
すなわち bucket 軸を row に畳んだ `[bucket*out, in]` で、tatara の bucketed weight
layout と一致する。

これに合わせ tatara も全 weight + 全 bias を対象にする:

| tensor | レイアウト | norm 粒度 |
|---|---|---|
| FT weight `[ft_in, ft_out]` | strided column | per-output-neuron (ft_out 個) |
| L1f weight `[ft_out, l1_out]` | strided column | per-output-neuron (l1_out 個) |
| L1 weight `[bucket*l1_out, ft_out]` | contiguous row | per-output-neuron |
| L2 weight `[bucket*l2_out, l2_in]` | contiguous row | per-output-neuron |
| L3 weight `[bucket*1, l2_out]` | contiguous row | per-output-neuron |
| 各 bias (FT/L1/L1f/L2/L3) | 1D | per-tensor scalar |

PSQT は reference の nnue-pytorch に存在しない層なので対象外。Simple アーキは
nnue-pytorch 対応物でないため `--norm-loss` を reject する (LayerStack 専用)。

### generic group indexing

3 レイアウトを `(n_groups, group_pitch, elem_stride, group_len)` の 4 値で統一
表現し、kernel を 1 対 (`norm_loss_reduce` + `norm_loss_apply`) に集約する:

```
off(g, pos) = g * group_pitch + pos * elem_stride   (g < n_groups, pos < group_len)
```

- contiguous row `[n_groups, group_len]`: `pitch=group_len, stride=1`
- strided column `[group_len, n_groups]`: `pitch=1, stride=n_groups`
- per-tensor scalar: `n_groups=1, pitch=0, stride=1, group_len=テンソル長`

`norm_loss_reduce` は 1 thread = 1 group で per-group L2 を `norm_scratch` に書き
(集計順は reference CPU と同一 pos 昇順)、`norm_loss_apply` は 1 thread = 1 weight
で補正を乗ずる。`norm_scratch` (長さ = 対象中の最大 group 数) は同 stream 上で
tensor ごとに再利用する (次 tensor の reduce 前に当該 apply が読み終える)。

### 適用順序 (radam step の前)

norm loss は radam optimizer step の **前** に適用する:

1. radam の per-layer 量子化 clamp が最後の演算になり clamp 不変条件を保つ
   (norm loss が clamp 後に微小に範囲外へ押し出すのを避ける)。
2. FT の FP16 mirror (`ft_w_h`) は後続 radam / lookahead が master から再同期する
   ため、norm loss 側で mirror を更新する kernel variant が不要になる。
3. nnue-pytorch の "decay/normloss を step に先立ち p.data に適用" する順序とも
   整合する。

既定 no-op の `--weight-decay` とは独立に併用可能。両者とも weight への乗法縮小だが、
decay は radam step 内 (`1 - decay*rate`)、norm loss はその前段。

## Alternatives considered

- **per-tensor (Frobenius) norm に統一**: kernel はレイアウト非依存で最も単純だが、
  Ranger21 の linear-layer 既定 (per-row) と異なり reference fidelity を欠く。
  採用せず。
- **対象を dense weight (L1/L2/L3) のみに最小化**: kernel surface は減るが、
  reference は FT / L1f / bias も含むため、"最適な状態を実験で確かめる" という
  目的に反する。採用せず (全層を対象に faithful port)。
- **radam step の後に適用 + FT mirror 専用 variant**: clamp 不変条件を崩し、
  mirror 同期 kernel が 1 種増える。前段適用で両方回避。

## Consequences

- opt-in 時、対象 weight に対し reduce + apply の 2 pass が増える。最大の FT
  weight (`ft_in * ft_out`) で per-step に追加 read 1 + read/write 1 pass。
  棋力検証用の opt-in flag なので throughput は許容範囲とする。
- 数値検証: CPU reference (`gpu_kernels::pointwise::norm_loss`) の unit test と、
  3 レイアウトを covering する GPU↔CPU 同等性テスト
  (`norm_loss_reduce_matches_cpu` / `norm_loss_apply_matches_cpu`)。
- 棋力可否は held-out 400 sb + SPRT で長期 run により判定する (baseline =
  現行ベスト ft-fanin / l1l2l3-fanin の勝者)。
- experiment.json に `norm_loss_factor` を記録 (有効時のみ) して再現性を確保。
